Definition — AES-256

What is AES-256?

AES-256 is the strongest variant of the Advanced Encryption Standard, standardised by NIST (2001). It uses a 256-bit key and runs 14 rounds of transformation. It secures classified government files, military communications, full-disk encryption, and financial systems worldwide.

Definition

Symmetric Block Cipher

The same 256-bit key encrypts and decrypts. Data is processed in fixed 128-bit (16-byte) blocks. AES-256 is the strongest AES variant.

Key Size

256-bit = 32 bytes

2256 possible keys — more than the number of atoms in the observable universe. No brute-force attack is feasible.

Rounds

14 Rounds

AES-256 applies 14 rounds of 4 operations (SubBytes → ShiftRows → MixColumns → AddRoundKey), each with a unique round key. Round 14 skips MixColumns.

Used In

Top-Secret Systems

US government classified data, military comms, BitLocker, VeraCrypt, Signal, AWS KMS, and hardware security modules.

Why 256, not 128? AES-128 is already unbreakable by today's computers. AES-256 provides an extra 128-bit security margin specifically to resist future quantum computers (Grover's algorithm halves the effective key size, so 256 → 128-bit post-quantum security).

Original Key (256 bits): Your 32-byte secret. Split into 8 words W[0]–W[7]. These form Round Key 0 directly. AES-256 needs 15 round keys total (60 words). The extra key length means AES-256 has its own unique step — the "Extra SubWord" — that AES-128 and AES-192 do not have.
How Many Rounds?

AES-256 Runs 14 Rounds — Here is Why

More rounds = more mixing = harder to break. AES-256 uses 14 rounds because its 256-bit key is larger, so more rounds are needed to fully "spread" the key influence across all bytes. The rule: Nr = Nk + 6, where Nk = key words. For AES-256: Nk=8, so Nr = 8+6 = 14.

Why exactly 14? Formula: Rounds = Key words + 6 = 8 + 6 = 14.
Each round takes the data one step further from the original. After 2 rounds, every output byte already depends on all 32 input bytes (the "avalanche effect"). 14 rounds makes reversal without the key computationally impossible.

Reference Table

S-Box — Substitution Lookup Table

Used in SubBytes: every byte is replaced by the value at S-Box[high nibble][low nibble]. This table is shown as a hint at the right moment during encryption — it will highlight when SubBytes runs.

How to read the S-Box: Take byte 0x53. High nibble = 5 → row 5. Low nibble = 3 → column 3. S-Box[5][3] = 0xED. The byte 0x53 becomes 0xED after SubBytes. Hover any cell above to confirm.
