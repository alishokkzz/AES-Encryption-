Definition

What is AES?

AES — Advanced Encryption Standard — is a symmetric block cipher standardised by NIST in 2001. It replaced DES and now secures HTTPS, Wi-Fi (WPA2/3), disk encryption, messaging apps, banking, and government systems worldwide.

Definition

What is AES?

A symmetric block cipher. It encrypts fixed 128-bit (16-byte) blocks using a secret key. Same key encrypts and decrypts.

Purpose

Why is it needed?

Without encryption, intercepted data is readable. AES-128 has 2128 possible keys — brute force is computationally impossible even with all computers on Earth.

Mechanism

How does it work?

Runs data through 10 rounds of four operations — SubBytes, ShiftRows, MixColumns, AddRoundKey — each using a unique round key derived from your secret.

Real-world use

Where is it used?

HTTPS/TLS, WPA2 Wi-Fi, WhatsApp, Signal, iPhone encryption, BitLocker, AWS, Azure, Bitcoin wallets, passports, military communications.

Simple explanation: Your message sits on a 4×4 grid of bytes. AES shuffles, replaces, and mixes those bytes 10 times using precise math. Without the key, reversing the process is computationally infeasible.

The Four Core Transformations

SubBytes — Non-linear Substitution

What: Every byte replaced via the AES S-Box lookup table. The high nibble (left hex digit) picks the row; the low nibble (right hex digit) picks the column. The table entry at that position becomes the new byte.
Why: Introduces non-linearity. Without it, AES is a linear function and breakable with algebra. The S-Box was designed to resist differential and linear cryptanalysis.

ShiftRows — Cyclic Row Rotation

What: Row 0 stays. Row 1 rotates left by 1. Row 2 rotates left by 2. Row 3 rotates left by 3.
Why: Ensures bytes from different columns end up in different columns so MixColumns can mix bytes from many original positions — this is called diffusion.

MixColumns — Column Diffusion

What: Each column treated as a polynomial in GF(2⁸) and multiplied by a fixed matrix. Each output byte depends on all 4 input bytes of that column.
Why: With ShiftRows, provides full diffusion — after 2 rounds every output byte depends on every input byte (avalanche effect). MixColumns is skipped in Round 10.

AddRoundKey — XOR with Round Key

What: Every byte of the state is XOR-ed with the corresponding byte of the round key. XOR (⊕) compares bits one by one: same → 0, different → 1.
Why: The only step that uses the secret key. Without this, AES produces the same output no matter what key you give it. XOR is also self-inverse: (A ⊕ K) ⊕ K = A, so decryption uses the exact same operation.
